Lesson 9 Notes

Lesson 19: Undestanding Dynamic Websites and HTML5 Applications

This lesson taught us that there are two types of scripting: client and server-side scripting. The type that we are focusing on in this lesson is client-side, which includes adding a bit of JavaScript to our sites to make them more interactive. As an example, the lesson showed us how to generate a random quote with JavaScript and later taught us how to make this code unobtrusive. It emphasized the importance of unobtrusive script so that all users could potentially access the information, even if their browser (or device) doesn’t support JavaScript.

The lesson also introduced the concept of the Document Object Model, which is a framework for referring to elements and their associated objects. This came very handy when we used the DOM to make the Quotable Quotes page unobtrusive, by calling the quote element with its id to HTML first, before using JavaScript to generate the quote. We also learned how the DOM helped us change the “featured” image in a rotation of smaller images below it by filling the source of the image with the smaller images when writing the specifying the event “onclick.” Finally, the lesson probed us to start thinking about how the foundations learned here could be used to build more complex HTML applications, especially when we start to think about how to use server-side scripting for larger projects.

What You'll Learn in this Lesson:

  • How to conceptualize different types of dynamic content
  • How to include JavaScript in your HTML
  • How to display randomized text with JavaScript
  • How to change images by using JavaScript and user events
  • How to begin thinking ahead to putting all the pieces together to create HTML5 applications

How to write comments in JavaScript:

Use two forward slashes // to comment code in JavaScript. Anything after the slashes will be only visible on the back-end.

Lesson 20: Getting Started with JavaScript Programming

This lesson dove a little deeper into JavaScript and its various components. We learned some basic concepts such as statements, functions, variables, and objects. JavaScript also supports conditional statements, loops, and events. We learned the order that scripts are executed and a few rules for JavaScript syntax. As we experiment and write code, the lesson emphasized the value of comments to help us understand the script when we need to go back to it or when reading someone else’s code.

Like HTML and CSS, JavaScript can be very helpful if we follow some best practices such as separating it into external files, using comments to explain our code, and being as general as possible so that all browsers can access JavaScript. We also learned the importance of keeping JavaScript optional so that anyone should be able to access our site (aka progressive enhancement). Finally, this lesson covered JSON, a common way of structuring and storing information used by JavaScript. Because JSON is very simple, easy to read, and largely self-documenting, it has become a common way to interchange data.

What You'll Learn in this Lesson:

  • How and why to organize scripts using functions
  • What objects are and how JavaScript uses them
  • How JavaScript can respond to events
  • How and when to use conditional statements and loops
  • How browsers execute scripts in the proper order
  • Basic syntax rules for avoiding JavaScript errors
  • What JSON is and how it can be used

Best Practices for JavaScript

  • Use comments liberally
  • Use a semicolon at the end of each statement and use only one statement per line
  • Use external JavaScript files whenever possible
  • Avoid being browser specific
  • Keep JavaScript optional

Lesson 21: Working with the Document Object Model (DOM)

This lesson uncovered more about the Document Object Model and how it structures a web browser and the documents within it. We learned that the window is the parent object for all objects and the child objects can include the document, location, history, and navigator objects. The lesson dove deeper into properties of each that help us name and refer to these objects in our scripts.

Then, this lesson taught us a bit more about the DOM structure. Although we were introduced to the DOM in the past, this lesson helped contextualized the DOM by showing us how it can represent the structure in an HTML document. We learned that each container or element in a document is called a node in the DOM and the relationship between parents, children, and siblings in the structure. The lesson covered a few basic node properties, node relationship properties, and document and node methods to customize and create dynamic sites. The lesson concluded with a demonstration of how to create and show/hide layers, move layers around, and modify/add the text in a page all using concepts and commands learned.

What You'll Learn in this Lesson:

  • How the W3C DOM standard makes dynamic pages easier to control
  • The basics of the standard DOM objects: window, document, history, and location
  • How to work with DOM nodes, parents, children, and siblings
  • How to access and use the properties of DOM nodes
  • How to access and use DOM node methods
  • How to control element positioning with JavaScript
  • How to hide and show elements with JavaScript
  • How to use JavaScript to add and modify text within a page
