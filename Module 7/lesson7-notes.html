Responsive Web Design Notes

Lesson 16: Understanding the Importance of Responsive Web Design

This lesson was all about creating responsive web pages that work across all browsers and most devices. By designing a site intended for mobile devices first, and making sure it scales to larger screens can ensure that your website is accessible to all, while providing a pleasant user experience. Not only that, but responsive web design streamlines the web development process because only one version of the code is needed. It forces you to keep organized, by emphasizing the use of clean code, separating style sheets, and using semantic code for the next time you need to go back and edit.

We were also introduced to progressive enhancement, which involves starting out with the most important information as a foundation and building its complexity so that the most basic browsers will at least see the basic information. The lesson reiterated the basic tags needed for a functioning webpage and recommended listing style properties with more basic features first, in case the browser canâ€™t interpret the modern features later down the style sheet. Finally, we learned the importance of validating your code so that you ensure that all your work can be seen and experienced correctly!

What You'll Learn in this Lesson:

  • What is responsive web design (RWD)
  • Why we need RWD
  • What is progressive enhancement
  • How to write HTML that's RWD ready

Lesson 17: Designing for Mobile Devices

This lesson covered mobile first design as a way to approach your web building process. It taught us how to design effective interfaces for mobile devices first by avoiding flash, configuring the viewport, creating flexible width designs and functional/visible elements so that our sites are not penalized by search engines. The lesson explained why mobile first is so important. More and more users are accessing websites via smaller devices which forces developers to streamline their content and think about what is most important to the mobile user.

To create elements, such as tables and elements, that are difficult to fully experience on a mobile device, the lesson covered some ways of creating responsive tables by using css to resize the cells, rearrange the table, and remove or hide content. Similarly, images can be set to flexible widths with css properties or within the html img tag. The lesson helped us determine when we might need to use entirely different images on mobile. Finally, we covered how to create responsive layouts without media queries by setting css columns. This allows you to generate a single column when the browser is small. Overall, responsive web design is very important, yet difficult to do. By focusing on mobile customers first and giving them the basic information, we ensure that we meet the needs of the majority of our website visitors.

What You'll Learn in this Lesson:

  • How mobile design differs from standard web design
  • What the Mobile First design philosophy is
  • How Mobile First is important to RWD
  • How to build responsive tables and images for mobile devices
  • How to use CSS columns to create responsive designs without media queries

Lesson 18: Using Media Queries and Breakpoints

This lesson was all about media queries and how to use them to further build on responsive web design. We learned that most rwd have at least one breakpoint, which defines where the design adjusts to accomodate different media types. By using media queries, which are logical expressions that define rules for what styles to use on specific device properties, we can better control how our sites look and function on different devices, browser sizes, and screen densities.

More specifically, this lesson taught us how to define breakpoints using media queries and best practices for queries. For example, when creating a style sheet, we should start with the most basic, unchanging styles first, then add specific styles for all devices, and finally the media queries to define styles for medium and larger devices. This ensures that we target mobile devices first and takes advantage of the cascading component of css. Finally, the lesson covered how to find the best breakpoints for your website, making sure that you do so when the page starts to break rather than creating a breakpoint for every possible device out there.

What You'll Learn in this Lesson:

  • How to write a CSS media query
  • How to use different media types
  • How to create media query expressions
  • Understanding logical keywords in your queries
  • What CSS breakpoints are
  • How to define breakpoints
  • How to find the best breakpoints for your website
