JavaScript Best Practices

This lesson introduced the importance of using JavaScript carefully and intentionally. We learned some of the best practices for using JavaScript which, ironically, involves using JavaScript as little as possible and ensuring that you don’t interfere with the accessibility of the content on your page. For example, by separating the content, presentation, and behavior of your webpage into three different files, you are better able to maintain a large site, reuse the same files for multiple pages of a site, and play each language to its strengths. JavaScript can take up a lot of space for browsers and not all will support JavaScript, so it is important to comply with web standards so that your site is not penalized or difficult to find.

To know more information about a user’s browser, we learned how to read and display browser information using the navigation object and then based on the features that each browser can support, we learned to use feature sensing to customize what is shown depending on the browser’s capabilities. This helps us avoid errors and create unobtrusive script. Finally, we learned some options for supporting older browser versions and even browsers that don’t support JavaScript at all by displaying a message or simply keeping JavaScript optional at all times. The lesson also covered an excellent example of an unobtrusive script where the page is always functional, even if JavaScript is not supported.

  • Best practices for creating unobtrusive scripts
  • The importance of separating content, presentation, and behavior
  • The importance of following web standards to create cross-browser scripts
  • How to read and display browser information
  • How to use feature sensing to avoid errors
  • How to support older, less compliant browser versions
  • How to support non-JavaScript browsers

Reflection on the Lesson

The lessons we learned in this chapter emphasized the importance of using JavaScript carefully and intentionally, not only so our webpages can be easier to manage, but so that all your content can be accessible. Not all browsers support JavaScript and even those that do may have JavaScript disabled. As a result, not everyone will be able to interact with your sites the way you can. By using JavaScript as little as possible, keeping your JavaScript file separately, and following broad web standards, we can ensure that most people can at least access the information on our page. This lesson was extremely important in getting us to think about the user’s point of view at all times and to get creative when balancing functionality and presentation.



Using Third-Party JavaScript Libraries and Functions

This lesson introduced third party libraries and frameworks for JavaScript code, which may lighten the load for developers. Although there are a few objections, in general third party libraries enable you to save time and energy, become more efficient, and ensure that your script will work on several browsers and different versions. The lesson listed a few common libraries such as jQuery and Prototype that help simplify tasks so you don’t have to reinvent the wheel every time. We learned the best ways to include this code to our websites, by using to reference the code and putting it near the end of our HTML doc so that the webpage loads first.

Then, the lesson introduced frameworks, which are more complex and larger, mostly meant to establish an architectural pattern on an application. We learned the three components of an application: the model, view, and controller, which allow the user more control when interacting with the page. The work to code these applications can be grueling and tedious, which is why the lesson recommended a few frameworks to handle some of that work for you. While it is important to try to understand what you are coding, especially as a beginner, taking advantage of helpful tools like libraries and frameworks is nonetheless a smart move.

Reflection on the Lesson

Third party script libraries and frameworks can be an extremely useful asset, especially for larger, more complex projects. They provide basic functionality and shortcuts so you can get complete pages or applications much quicker and focus on the more important aspects of your page, such as the user interface and designing the experience. I am grateful that the book covered this lesson, not to replace all our existing knowledge of JavaScript, but to supplement it so that we could become more efficient developers and use the various tools available to continue to improve.
