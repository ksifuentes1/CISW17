Using JavaScript Variables, Strings, and Arrays

Summary:

    This lesson dove deeper into JavaScript, explaining how variables, functions, methods, strings, arrays, and objects work. First, we learned how to declare and use variables, assigning values to them, and how JavaScript reads the values in a prescribed order of operations. Within a variable, you can have string objects which are groups of characters or text surrounded by quotation marks. There are a few handy tools to work with strings, such as concatenating, calculating the length of a string or converting strings to all caps. Also, we learned how to find specific items in a string by using indexOf method.

    Besides variables and strings, this lesson covered arrays, which are groups of items (numeric or not) treated as a single unit. An array can also include string of text which can be sorted, split, joined, or counted. For grouping various JavaScript statements, we were taught how to define functions with the correct syntax and then how to call the function so that it executes. Then, we learned more about objects, which can store several properties, compared to a variable that can only store one piece of information. This came in handy when we learned how to develop a contact card with a person’s name, email, address, and phone number.

    Finally, the lesson covered a few numerical capabilities by using the Math object’s methods. For example, we can round numbers and generate random numbers for a variety of purposes. JavaScript also has a Date object that lets us call and view specific dates and times, across several time zones. All are very useful and necessary tools to know when working with JavaScript.

What You'll Learn in This Lesson

  • How to name, declare, and assign variables
    • Variable names can include letters of the alphabet, both upper- and lowercase. They can also include the digits 0–9 and the underscore (_) character.
    • Variable names cannot include spaces or any other punctuation characters.
    • The first character of the variable name must be either a letter or an underscore.
    • Variable names are case sensitive; for example, totalnum, Totalnum, and TotalNum are interpreted as separate variable names.
    • There is no official limit on the length of a variable name, but it must fit on one line. Frankly, if your variable names are longer than that—or even longer than 25 or so characters, you might consider a different naming convention.
  • How to choose whether to use local or global variables
    • Defining Global and Local Variables
      • Global Variables have the entire script (and other scripits in the same HTML) and can be used anywhere, even within functions.
      • Local Variables have a single function as their scope and can only be used within the function they are created in.
    • Assigning Value to Variables
      • Use the = sign to assign a value to a variable like so: var lines = 40;
    • How to Increment and Decrement Variables
      • Use the += sign to add a number from a variable like so: lines +=1; or use ++ to add 1 to the variable like so: lines++;
      • Use the -= sign to subtract a number from a variable like so: lines -=1; or use -- to subtract 1 to the variable like so: lines--;

    • String Object: stores a group of text characters and are named similarly to other variables
  • How to create and use arrays of numbers and strings
    • Array: a numbered group of data items that you can treat as a single unit. An array can contain strings, numbers, objects, or other types of data.
  • How to define, call, and return values from functions
    • Function: A function is a group of JavaScript statements that can be treated as a single unit
    • To return a value from a function: begin with the function keyword, the function's name, and the parameters it accepts. Then use the return keyword along with the variable that the function created and end the code with a semicolon and closing curly braces.
  • How to use object properties and values
    • Object: an object is a more complex standalone entity that can store multiple properties (aka variables with values).
